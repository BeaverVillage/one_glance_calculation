# 한눈체크 운영 정리 완료 기록

## 1차 운영 정리

- 홈페이지 자주 쓰는 기능을 5개로 고정했습니다.
- 운영 안정성이 낮은 기능은 공개 메뉴, sitemap, 데이터 출처, 환경변수 예시에서 제외했습니다.
- 외출 위험 종합 체크의 기상특보 보조 조회는 실패해도 핵심 결과를 가리지 않도록 정리했습니다.
- 응급실·야간 병원 화면의 개발용 표시 문구를 조회 전 안내 문구로 교체했습니다.

## 2차 지도형 UI 기반 정리

- 응급실·야간 병원·약국 확인 페이지를 지도형 포맷으로 재정렬했습니다.
- 상단 조건 카드, 지도, 결과 목록, 선택 상세 패널이 한 화면 흐름으로 보이도록 구성했습니다.
- 별도 사이드 안내 패널을 줄이고 중요한 안내는 상단 한 줄과 하단 안내 목록으로 이동했습니다.
- `assets/js/kakao-place-link.js`를 추가해 카카오맵 바로가기/검색 fallback 공통 구조를 준비했습니다.

## 3차 의료기관 카카오 캐시

- 의료기관 카카오맵 place_url 로컬 캐시 구조를 추가했습니다.
- `assets/data/medical/kakao-place/` 지역별 카카오 장소 캐시를 사용합니다.
- `scripts/enrich-medical-kakao-places.js`로 응급실, 야간 병원, 야간 약국의 카카오 장소 매칭 캐시를 생성할 수 있게 했습니다.
- 의료기관 상태 정보는 캐시하지 않고, 장소 링크 정보만 캐시합니다.

## 4차 응급실 지도형 UI 최종 정리

- 응급실·야간 병원·약국 확인 페이지를 기존 지도형 도구 흐름에 맞춰 최종 재구성했습니다.
- PC에서는 왼쪽 조건 패널, 오른쪽 지도·결과 목록·선택 상세 패널 구조로 정리했습니다.
- 모바일에서는 조건, 지도, 결과 목록, 상세 패널 순서로 자연스럽게 내려가도록 정리했습니다.
- 카카오맵 로컬 캐시가 있으면 `카카오맵 바로가기`, 없거나 신뢰도가 낮으면 `카카오맵 검색`으로 표시합니다.
- 큰 안내 카드와 과한 상태 카드를 줄이고, 실제 행동에 필요한 전화·지도·상세 확인 흐름을 우선 배치했습니다.

## 5차 최종 QA

- 메인 페이지 자주 쓰는 기능 5개 유지 여부를 확인했습니다.
- 공개 메뉴, sitemap, 데이터 출처, 환경변수 예시에 불필요한 기능 흔적이 남지 않도록 정리했습니다.
- 응급실·야간 병원·약국 지도형 UI의 모바일 단일 컬럼 흐름을 확인했습니다.
- 외출 위험 보조 오류 문구가 과도하게 노출되지 않도록 확인했습니다.
- 주요 JavaScript, Cloudflare Functions, JSON, sitemap, HTML 파싱을 점검했습니다.
