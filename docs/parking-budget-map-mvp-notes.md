# 주차비 계산 지도 MVP 메모

## 최종 MVP 방향

이번 MVP는 길안내 기능이 없는 카카오맵 기반 주차비 계산 지도다.

핵심 범위:

- 카카오맵 JavaScript API로 지도와 마커 표시
- 카카오 Local API로 목적지 검색과 좌표 변환
- 공공데이터 기반 주차장 후보 조회 구조
- 목적지 좌표와 주차장 좌표 사이의 직선거리 계산
- 입차/출차 시간, 빠른 시간 선택, 할인 조건 기준 예상 주차비 계산
- 일주차 전환점, 실시간 빈자리, 만차 위험도, 데이터 신뢰도 표시
- 모바일 바텀시트 형태의 추천 결과 UI

## 이번 MVP에서 제외한 항목

- 사용자 출발 위치 입력 기능
- 자동차 경로 계산
- 차량 이동 시간 계산
- 차량 이동 거리 계산
- 유류비/통행료 계산
- 외부 경로 API 연동
- 차량 시간 기준 정렬
- 외부 지도·길안내 서비스 연동

제외 이유: 경로 API는 비용, 약관, 호출량 제한 리스크가 있으므로 이번 MVP는 주차비 계산과 지도 표시 자체에 집중한다.

## 필요한 환경변수

- `KAKAO_MAP_JS_KEY`: 지도 표시용 JavaScript Key
- `KAKAO_REST_API_KEY`: 카카오 Local API 목적지 검색용 REST Key. Functions에서만 사용
- `PUBLIC_DATA_API_KEY`: 공공데이터포털 주차장/특일 정보 연동용
- `SEOUL_OPEN_API_KEY`: 서울 열린데이터 주차장/실시간 빈자리 연동용
- `HOLIDAY_API_KEY`: 한국천문연구원 특일 정보 공휴일 판정용

## API 라우트

- `GET /api/parking/places?q=검색어`
- `GET /api/parking/lots?lat=...&lng=...&radius=...`
- `POST /api/parking/estimate`
- `GET /api/parking/realtime?parkingLotId=...`
- `POST /api/parking/recommend`
- `GET /api/parking/holidays?date=YYYY-MM-DD`

## 추천 점수

추천점수는 다음 요소만 사용한다.

- 요금점수: 할인 후 주차비가 낮을수록 높음
- 거리점수: 목적지와 직선거리가 가까울수록 높음
- 만차위험점수: 위험도가 낮을수록 높음
- 데이터신뢰도점수: 신뢰도가 높을수록 높음
- 공영가산점: 공영 후보에 소폭 가산

정렬 옵션:

- 추천순
- 저렴한순
- 가까운순
- 빈자리순
- 신뢰도순

## 데이터 표시 원칙

- 실시간 가능 대수는 제공기관 데이터 기준이며 현장 상황과 다를 수 있다.
- 할인 계산은 선택한 할인율을 적용한 참고 계산이다.
- 목적지와 주차장 사이의 거리는 좌표 기반 직선거리다.
- 실제 요금, 운영시간, 할인 조건, 주차 가능 여부는 현장 또는 관리기관 안내를 확인해야 한다.

## 2026-06-17 반경 필터/하이브리드 fallback 수정

- `/api/parking/lots`와 `/api/parking/recommend`는 `radius`를 meter 단위로 해석한다.
- 최종 반환 전에도 `distanceFromDestinationKm * 1000 <= radius` 안전 필터를 한 번 더 적용한다.
- 공공데이터 또는 서울 열린데이터 호출이 성공했더라도 반경 내 후보가 0개이면 원거리 후보를 반환하지 않는다.
- 반경 내 공공데이터 후보가 부족한 경우 로컬 샘플 후보를 같은 반경 필터로 확인해 hybrid/fallback 후보로 사용한다.
- API 응답에는 `stats.normalizedCount`, `stats.withCoordinateCount`, `stats.nearbyCount`, `stats.sampleNearbyCount`, `stats.returnedCount`, `fallbackReason`을 포함한다.
- 사용자 UI에는 raw JSON을 노출하지 않고 “계산 가능한 주차장을 찾지 못했습니다. 검색 반경을 넓혀보세요.”처럼 짧은 안내만 표시한다.

## 2026-06-17 대시보드형 UI 적용

데스크톱에서는 다음 구조를 사용한다.

- 상단 전체 폭: 1단계 목적지, 2단계 방문 일정, 3단계 차량/할인 조건
- 하단 왼쪽: 4단계 추천 결과 스크롤 패널
- 하단 오른쪽: 카카오맵 지도

모바일에서는 입력 카드, 지도 카드, 추천 결과 카드, 설명/FAQ 순서의 1열 레이아웃으로 전환한다.

## 2026-06-17 대시보드형 UI 재개편

기능 로직은 유지하고 화면 구조만 다음 흐름으로 재정리했다.

- 상단 전체 폭: 1단계 목적지 검색, 2단계 주차 일정, 3단계 차량/할인·필터·정렬
- 하단 왼쪽: 4단계 추천 결과 스크롤 패널
- 하단 오른쪽: 카카오맵 지도 패널

데스크톱에서는 추천 결과와 지도를 같은 행에 배치해 지도 아래에 큰 빈 공간이 생기지 않도록 했다. 추천 카드가 많아져도 페이지 전체가 과도하게 길어지지 않도록 하단 왼쪽 추천 결과 패널 내부에서만 스크롤되게 구성했다.

모바일에서는 2열 구조를 강제하지 않고 입력 카드, 지도 카드, 추천 결과 카드, 설명/FAQ 순서의 1열 구조를 유지한다. 출발지·길찾기·차량 이동시간 기준 정렬 문구는 계속 제외한다.

## 2026-06-17 UI/데이터 안정화 추가 메모

이번 산출물은 `hannuncalc_parking_calc_map_dashboard_layout_final.zip` 기준으로 주차비 계산 지도 페이지의 UI 완성도와 데이터 fallback 구조를 보강했다.

- 긴 법적/데이터 안내 박스는 입력/추천 흐름을 방해하지 않도록 지도 아래 compact notice로 이동했다.
- 추천 결과 상단 문구는 `추천 결과입니다.` 중심으로 줄이고, 데이터 상태는 짧은 배지로 분리했다.
- 데스크톱 추천 결과 패널 높이를 확대하고, 추천 카드 목록은 패널 내부에서만 스크롤되도록 조정했다.
- 지도 이동 후 자동 API 호출을 남발하지 않도록 `현재 지도에서 주차장 다시 찾기` 버튼을 추가했다. 지도 idle 후 700ms debounce, 이전 검색 중심에서 약 500m 이상 이동하거나 줌 레벨이 의미 있게 바뀐 경우에만 버튼을 표시한다.
- 전국주차장정보표준데이터가 주변 검색 API가 아니라는 한계를 보완하기 위해 전국 주차장 캐시 파일을 추가했다.
  - `functions/api/parking/_data/national-parking-lots.json`
  - `assets/data/parking/national-parking-lots.json`
- `fetchPublicParkingLots`는 더 이상 1페이지만 고정 조회하지 않고, 기본 3페이지까지 조회하도록 했다. 운영 환경에서는 별도 수집/캐시 파이프라인 또는 D1/JSON 캐시가 더 적합하다.
- 부산역, 대구역, 광주송정역, 대전역, 제주공항 주변 캐시 후보를 추가해 서울 외 지역에서 서울 샘플이 섞이지 않도록 했다.
- 서울시 실시간 주차 가능 대수는 서울 후보에 매칭되는 경우만 참고하고, 서울 외 지역은 기본적으로 `실시간 정보 없음`으로 표시한다.
- 이번 MVP 범위 밖의 경로·출발 위치·이동 시간 관련 기능은 계속 제외한다.
